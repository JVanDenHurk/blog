TryHackMe Cyber Security 101 Certification

The Cyber Security 101 pathway provided a beginner-friendly introduction to the core areas of computer security. It covered essential topics including computer networking, cryptography, system basics in Windows, Linux, and Active Directory, as well as offensive and defensive security tools.

Windows and AD Fundamentals

Gained foundational knowledge of the Windows operating system and how Active Directory (AD) is used to manage users, computers, and permissions in a networked environment. Key topics included:

  • Windows OS Basics
    Understanding the Windows file system, command line (CMD and PowerShell), user roles, and system tools.
  • Active Directory
    Exploring domain controllers, organizational units (OUs), users, groups, and group policy management.
  • Windows Security
    Learning about access control, user authentication, and common security configurations in a Windows environment.

Command Line

Gained comprehensive hands-on experience with the Linux shell and Windows PowerShell, learning how to interact with operating systems, automate tasks, and analyze system behavior. Key topics included:

  • Linux Shell Basics
    Introduction to Linux command-line interfaces (CLIs), including bash and other shell types. Practiced using commands to navigate, manipulate files, and execute scripts.
  • System & Network Information
    Retrieved system info using commands like uname, top, ps, ifconfig/ip, and used tools like ping, netstat, and traceroute for basic network troubleshooting.
  • File & Disk Management
    Learned to manage files, directories, and disk space using tools such as df, du, lsblk, and mount.
  • Process & Task Management
    Monitored and controlled system processes using ps, top, kill, and job control commands like bg and fg.
  • Shell Scripting
    Gained exposure to basic shell scripting concepts—variables, loops, conditionals, and writing practical scripts like the "Locker Script" for automation.
  • PowerShell Fundamentals
    Explored Windows PowerShell, learning how to work with cmdlets, retrieve system and network data, and perform file system operations.
  • Piping, Filtering, and Sorting
    Mastered data manipulation by chaining commands using pipes and cmdlets like Sort-Object, Where-Object, and Select-Object in PowerShell.
  • Real-Time Analysis & Automation
    Used PowerShell for live system monitoring, real-time process analysis, and wrote scripts to automate repetitive administrative tasks.

Networking Fundamentals

Networking is the backbone of cybersecurity, and this section gave me a strong foundation in how devices communicate, how data travels, and how we can analyze and secure that communication. From basic models to real-world tools, I explored how everything connects.

Networking Concepts

  • OSI & TCP/IP Models
    Got to grips with how data moves through different layers of a network, from physical wires all the way up to applications.
  • IP Addresses & Subnets
    Learned how devices are assigned unique addresses and how subnetting helps organize and secure networks.
  • TCP vs UDP
    Explored the differences between reliable (TCP) and fast (UDP) protocols and when to use each.
  • Encapsulation
    Understood how data is packaged and passed through layers to reach its destination.
  • Telnet
    Used Telnet to connect to remote machines and saw how insecure protocols work under the hood.

Networking Essentials

  • DHCP
    Discovered how devices automatically receive their IP settings when they join a network.
  • ARP
    Learned how IP addresses are linked to physical MAC addresses so devices can talk locally.
  • ICMP
    Used tools like ping and tracert to diagnose network issues and understand how packets are routed.
  • Routing & NAT
    Explored how routers direct traffic between networks, and how NAT translates private IPs to public ones.

Core Networking Protocols

  • DNS
    Understood how human-friendly domain names get translated into IP addresses computers can use.
  • WHOIS
    Used WHOIS lookups to gather information about domain ownership and registration details.
  • HTTP/HTTPS
    Learned how websites are served over the internet and how HTTPS protects web traffic.
  • FTP, SMTP, POP3, IMAP
    Explored older protocols for file transfer and email, including how and why they can be insecure if not encrypted.

Secure Networking Protocols

  • TLS & HTTPS
    Dived into how encryption secures web traffic, and how TLS keeps data safe in transit.
  • SSH
    Used SSH to securely connect to remote systems via the command line — a must-have skill for sysadmins and pentesters.
  • SFTP, FTPS, SMTPS, POP3S, IMAPS
    Looked at secure versions of file and email transfer protocols, and how encryption is layered on top of legacy systems.
  • VPNs
    Explored how VPNs create encrypted tunnels for safe browsing and private communication, even over public Wi-Fi.

Wireshark: The Basics

  • Capturing Packets
    Used Wireshark to capture live traffic and understand how data looks as it flows through a network.
  • Packet Dissection
    Dug into individual packets to identify protocols, payloads, and potential issues.
  • Filtering Traffic
    Applied filters to zero in on specific conversations or protocols — super helpful when troubleshooting.

Tcpdump: The Basics

  • Command-Line Capture
    Learned to capture network traffic using just the terminal, a lighter alternative to Wireshark.
  • Filtering Expressions
    Used precise filter syntax to capture only the traffic I wanted to analyze.
  • Display Options
    Customized output formats to make it easier to interpret results directly from the CLI.

Nmap: The Basics

  • Host Discovery
    Scanned for live systems in a network to map out the environment.
  • Port Scanning
    Identified open ports to figure out what services are running — crucial for vulnerability assessments.
  • Version Detection
    Pulled back banner information and software versions to evaluate risk and exposure.
  • Scan Timing & Output
    Tuned scan speeds and output formats for stealth or performance depending on the situation.

Cryptography

In this module, I built a solid foundation in modern cryptography. I learned how symmetric and asymmetric encryption algorithms work, and why they're essential for secure communication. I explored real-world applications like TLS, SSH, and digital signatures, and gained a deep understanding of how hashing protects data integrity and passwords. I also got hands-on experience using tools like John the Ripper to crack password hashes.

Cryptography Basics

  • Why Cryptography Matters
    Understood the crucial role cryptography plays in safeguarding digital communication and protecting sensitive data.
  • Plaintext to Ciphertext
    Learned how data is transformed from readable formats into encrypted strings, making it unreadable to unauthorized users.
  • Historical Ciphers
    Explored classic encryption methods like the Caesar cipher and substitution ciphers to see how early encryption worked.
  • Types of Encryption
    Compared symmetric vs asymmetric encryption, including how keys are shared and managed securely.
  • Basic Math
    Brushed up on the math concepts behind encryption, like prime numbers and modular arithmetic.

Public Key Cryptography Basics

  • While doing this module I decided to have some fun and build a Caeser Cypher
  • Asymmetric Encryption
    Discovered how public and private keys enable secure communication without sharing secrets in advance.
  • RSA & Diffie-Hellman
    Learned how these two core algorithms work to encrypt and securely exchange data.
  • SSH
    Used SSH for secure remote access and saw how cryptography ensures safe terminal connections.
  • Digital Signatures & Certificates
    Understood how they ensure data authenticity and integrity in systems like HTTPS and email encryption.
  • PGP & GPG
    Explored how Pretty Good Privacy helps encrypt files and emails, and how GPG is used in practice.

Hashing Basics

  • What is Hashing?
    Learned how hash functions convert data into fixed-length values that uniquely represent the original input.
  • Storing Passwords Securely
    Saw the difference between insecure and secure password storage, including the use of salts and algorithms like bcrypt.
  • Recognizing Hash Types
    Practiced identifying different hash formats like MD5, SHA-1, and SHA-256.
  • Cracking Hashes
    Used tools to attempt cracking weak hashes and learned how attackers target poor password practices.
  • Hashing for Integrity
    Explored how hashes are used to verify file integrity and detect tampering.

John the Ripper: The Basics

  • Introduction to John
    Got started with John the Ripper, a powerful password cracking tool used by security professionals and pentesters.
  • Hash Cracking
    Cracked basic hashes and moved on to more advanced examples like Windows and Linux system hashes.
  • Real-World Scenarios
    Learned how to crack password-protected archives and SSH keys, and explored John’s different cracking modes.
  • Custom Rules
    Used custom rules to enhance cracking performance and efficiency.

Web Hacking

The Web Hacking module introduced me to the inner workings of web applications and the common vulnerabilities they face. I explored core web technologies like HTTP, JavaScript, and SQL, learned how to test them using Burp Suite, and gained insights into real-world threats through the OWASP Top 10 framework.

Web Application Basics

  • HTTP & URLs
    Learned how clients and servers communicate over HTTP, and how URLs map to resources on the web.
  • Request Methods & Status Codes
    Explored common methods like GET, POST, PUT, DELETE and response codes (200, 404, 500) to understand success, redirects, and errors.
  • Headers & Cookies
    Understood how HTTP headers control caching, content type, authentication, and how cookies maintain state between requests.
  • Request/Response Lifecycle
    Traced the full flow of an HTTP request from browser to server and back, including DNS lookup, TCP handshake, and payload delivery.

JavaScript Essentials

  • DOM Manipulation
    Learned to traverse and modify the Document Object Model to dynamically update page content and structure.
  • Event Handling
    Explored how to capture and respond to user interactions like clicks, keypresses, and form submissions.
  • XSS Vulnerabilities
    Understood how improper input sanitization can lead to cross-site scripting and how to prevent it.
  • Asynchronous JS
    Practiced using callbacks, promises, and async/await to handle network requests and improve user experience.

SQL Fundamentals

  • Basic Queries
    Mastered SELECT statements to retrieve data, including filtering with WHERE and sorting with ORDER BY.
  • CRUD Operations
    Learned to CREATE, UPDATE, and DELETE records, managing data lifecycle within relational tables.
  • Joins & Relationships
    Explored INNER JOIN, LEFT JOIN, and RIGHT JOIN to combine data from multiple tables based on key fields.
  • SQL Injection Risks
    Understood how unsanitized inputs can allow attackers to manipulate queries, and practiced using parameterized statements to defend.

Burp Suite: The Basics

  • Introduction to Burp Suite
    Discovered the core modules: Proxy, Scanner, Intruder, Repeater and why Burp is essential for web pentesting.
  • Installing & Configuring
    Set up Burp Community Edition and configured your browser to route traffic through the Burp proxy.
  • Proxying Requests
    Intercepted HTTP(S) traffic to analyze and modify requests and responses on the fly.
  • Site Map & Scanning
    Built a site map of discovered endpoints and launched passive/active scans to identify common vulnerabilities.

OWASP Top 10 – 2021

  • A01:2021 – Broken Access Control
    Learned how missing or misconfigured access controls allow attackers to act as other users or elevate privileges.
  • A02:2021 – Cryptographic Failures
    Understood the risks of weak or missing encryption and how sensitive data can be exposed in transit or at rest.
  • A03:2021 – Injection
    Explored SQL, OS, and LDAP injection techniques, and practiced input validation to prevent malicious payloads.
  • A04:2021 – Insecure Design
    Examined how design flaws and missing security controls can lead to exploitable logic errors.
  • A05:2021 – Security Misconfiguration
    Discovered common misconfigurations—default credentials, unnecessary services, error messages—and how to lock down systems.
  • A06:2021 – Vulnerable & Outdated Components
    Learned to inventory third-party libraries and frameworks, scanning for known CVEs and applying timely patches.
  • A07:2021 – Identification & Authentication Failures
    Understood how poor session management and weak credential policies can lead to account compromise.
  • A08:2021 – Software & Data Integrity Failures
    Explored risks from trusting unverified code or data updates, including supply-chain attacks.
  • A09:2021 – Security Logging & Monitoring Failures
    Learned why comprehensive logging, alerting, and monitoring are critical for detecting and responding to breaches.
  • A10:2021 – Server-Side Request Forgery (SSRF)
    Practiced identifying SSRF flaws that allow attackers to make unauthorized requests from the server’s context.
